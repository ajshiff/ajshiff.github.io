Canvas Quiz Data Reporter

35 hours,  January 2019

This tool generates an actionable report that identifies broken quiz questions in the university's LMS. In Winter 2019 semeter, many university quiz questions were broken with a variety of bugs that rendered the question unanswerable. This program downloads all quiz question data, scans the content for these broken qualities, and generates a report. This helped the university diagnose the scope of the issue, and decide the best course of action. I built the program to have one configurable file that would define all scan sequences and patterns to run on the quizzes. This allowed for definitions to be easily updated or added as new issues were found.

Canvas Everything Finder

52 hours,  October 2018 - November 2018

I improved the memory efficiency of our team's Canvas-link-finder tool by 93%. I also generalized the tool to be able to do more than search for links. When put up to the task of searching over 15,000 courses for specific details of items, it only used 600MBs of memory at any given time, compared to the 9GBs the previous tool required. It would run searches on 5000 courses in parallel. I reduced the memory footprint by searching the response data as it is recieved, and only keeping the match data so that irrelevant data would fall out of scope at the earliest point possible.

Mass Maintainance of Pathway Worldwide Resources

112 hours,  July 2018 - August 2018

I saved Pathway Worldwide thousands of dollars by programatically transforming over 7,000 csvs' content from one format into their new updated format. One of the most difficult parts of this project was editing their HTML snippets to follow their new format and styling standards. This was difficult because many of their HTML snippets each followed different standards. The algorithm I defined worked for 99% of their files, leaving very few that needed to be fixed by hand. The description of the project was updated by the stakeholder almost daily. If I were to redo this project, I would have taken a more agile approach to development by having definition modules, which would have kept selector logic seperate from core-alteration logic, thus saving time in the long run.
